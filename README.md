# MedControl — контент документации

Репозиторий со статьями (Markdown), изображениями и служебными файлами для сайта документации MedControl.

Корень репозитория повторяет каталог `docs-app/public/content/` в движке: при сборке сюда должно быть стыковано то же дерево, что отдаётся как `/content/...` на сайте.

## Документация в этом репозитории

| Файл | Назначение |
|------|------------|
| [docs/MIGRATION-PLAN.md](docs/MIGRATION-PLAN.md) | Пошаговый план разделения движок/контент, чек-листы, риски, целевая структура |
| [docs/CHAT-TRANSCRIPT.md](docs/CHAT-TRANSCRIPT.md) | Полный архив переписки с Cursor (решения и пояснения не теряются при очистке монорепо) |
| [references/README.md](references/README.md) | Исходники HTML для конвертации в статьи, палитра, медиа и референсы (не публикуются как страницы сайта) |

Имеет смысл **не дублировать** эти материалы в репозитории движка: держите их здесь и при необходимости дайте в движке одну ссылку на этот репозиторий.

## Связь с движком

Пока движок (`medcontrol-docs-platform` или аналог) может использовать свою копию `public/content/`. Источник правки контента для долгосрочной работы — **этот** репозиторий. Подключение к движку: submodule в `docs-app/public/content` или копирование контента в CI перед `npm ci` / `build` (см. план миграции).

После настройки submodule/CI дублирующие файлы контента в репозитории движка можно удалить, оставив только механизм подстановки и заглушку при необходимости.

## Базовая линия «до миграции»

Снимок проверки внутренних ссылок и списка путей из `flatArticles` на момент разделения зафиксирован в **репозитории движка**: файл `baseline-links-before-migration.txt`. При полном переносе артефактов можно добавить копию в `docs/`, если нужна одна точка хранения — сейчас каноничен файл в движке.

## Локальный Git и GitHub

Репозиторий можно вести только локально; для команды и CI добавьте remote:

```bash
git remote add origin <URL-репозитория-на-GitHub>
git push -u origin main
```
